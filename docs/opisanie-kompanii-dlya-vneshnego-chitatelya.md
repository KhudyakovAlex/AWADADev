# Компания: формальное описание

> Полное и достаточно формальное описание компании для человека без знания внутренних ID.  
> Пользовательский интерфейс к базе фактов: `docs/struktura-kompanii.md`.

## 1. Общая характеристика

| Параметр | Описание |
|----------|----------|
| Профиль деятельности | Автоматизация инженерных систем зданий, сооружений и территорий |
| Основной бизнес | Реализация проектов автоматизации объектов |
| Роль разработки ПО | Критически важная часть общего производственного контура |
| Масштаб | Более 50 сотрудников; в профильной модели детализированы основные подразделения численностью около 35 человек |

## 2. Основные направления деятельности

### 2.1. Проектный контур

Основной контур деятельности компании.

Состав работ:
- определение состава системы управления для объекта;
- адаптация решений под конкретный проект;
- разработка и использование аппаратной части;
- разработка и использование программной части;
- пуско-наладка на объекте;
- использование внутренних инженерных инструментов.

### 2.2. Продуктовый контур

Отдельное направление развития.

Состав:
- специализированное устройство;
- мобильное приложение пользователя для работы с этим устройством.

Продуктовый контур развивается отдельно от типовой проектной системы.

## 4. Продукты, решения и инструменты

### 4.1. Проектная система управления

| Компонент | Назначение |
|-----------|------------|
| Система управления на объекте | Комплексное решение для автоматизации объекта |
| Аппаратная часть | Устройства, обеспечивающие управление и взаимодействие с оборудованием |
| Программная часть | Серверные и клиентские программные компоненты системы |

### 4.2. Аппаратная часть

| Элемент | Назначение |
|---------|------------|
| Центральный контроллер | Базовое устройство системы |
| Модули расширения | Подключение и расширение функций системы |
| Датчик присутствия и освещенности | Сбор данных для сценариев управления |
| Настенная кнопочно-сенсорная панель | Пользовательское управление |
| Другие модули расширения | Дополнительные аппаратные компоненты семейства |

Примечание: прошивки рассматриваются как часть аппаратных изделий.

### 4.3. Программная часть

| Компонент | Назначение | Среда выполнения | Ответственный отдел |
|-----------|------------|------------------|---------------------|
| Серверная часть | Логика и обслуживание системы | Контроллер или отдельный Linux-компьютер | Основной отдел разработки ПО |
| Клиентская часть | Интерфейс работы пользователя | Рабочие станции и мобильные устройства | Основной отдел разработки ПО |
| Альтернативная веб-ветка | Частичное функциональное замещение основной ветки и развитие нового функционала | Веб-среда | Отдел прикладных веб- и мобильных решений |

### 4.4. Внутренние инструменты

| Инструмент | Назначение | Пользователи | Ответственный отдел |
|------------|------------|--------------|---------------------|
| Основной внутренний инструмент | Поддержка проектных и инженерных работ | Проектный и инженерный отделы | Основной отдел разработки ПО |
| Альтернативный внутренний инструмент | Альтернативная реализация на веб-стеке | Проектный и инженерный отделы | Инструментальный отдел разработки ПО |
| BIM-платформа | Подготовка решений на основе BIM-моделей зданий | Профильные подразделения разработки и проектов | BIM-направление разработки ПО |

### 4.5. Отдельный продуктовый контур

| Компонент | Назначение | Ответственный отдел |
|-----------|------------|---------------------|
| Специализированное устройство | Аппаратная основа продуктового решения | Отдел разработки устройств |
| Мобильное приложение пользователя | Работа пользователя с устройством | Отдел прикладных веб- и мобильных решений |
