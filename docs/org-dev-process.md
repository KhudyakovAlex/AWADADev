## 0. Об организации

> Разработка ПО — не единственное и не главное направление. Сначала фиксируем бизнес-контекст.

### 0.1. Чем занимается организация
- **Название / отрасль:** автоматизация управления инженерными системами зданий, сооружений, территорий
- **Конечная цель бизнеса:** реализация проектов автоматизации объектов
- **Основной вид деятельности:** проектное внедрение автоматизации + собственное производство части оборудования + OEM
- **Продукты / услуги для клиентов:**
  - проекты автоматизации объектов (здания, сооружения, территории)
  - комплектные шкафы с автоматикой и контроллерами
  - датчики присутствия и освещённости (частично свои)
  - сервер управления системами на объекте + клиентские (фронт) приложения
- **Не наше:** конечные исполнительные устройства (светильники, вентустановки и пр.)
- **Масштаб:** 50+ сотрудников

### 0.2. Как устроен проект (цепочка ценности)

```
Заказчик / объект
    → Проектный отдел + Инженерный отдел (конкретный проект)
    → Оборудование: своё производство и/или OEM
         • шкафы комплектные с автоматикой, контроллеры
         • датчики присутствия / освещённости (частично свои)
         • конечные устройства (свет, вентиляция…) — чужие
    → ПО основного отдела разработки: сервер на объекте + клиенты + протоколы/драйверы к оборудованию
```

### 0.3. Направления / подразделения (по роли в бизнесе)

| # | Подразделение / направление | Роль в бизнесе | Своё ПО / программисты |
|---|----------------------------|----------------|------------------------|
| 1 | Проектный отдел | ведёт проекты автоматизации объектов | — |
| 2 | Инженерный отдел | инженерия на проекте (вместе с проектным) | — |
| 3 | Отдел разработки новых продуктов | своё оборудование (шкафы, контроллеры, датчики…) | да, ~3 чел. (ближе к железу) |
| 4 | Основной отдел разработки ПО | сервер управления, фронт-клиенты, работа с оборудованием/протоколами | да, ~10 чел. |
| 5 | Параллельные малые отделы (предметные) | vibe-coding инструментов / дизайн UI | да, 1–2 чел. каждый |

### 0.4. Роль ИТ и разработки ПО в компании
- **Зачем нужна разработка:**
  - **Основной отдел ПО:** сервер, управляющий системами на объекте; клиентские приложения; механизмы обмена с разными видами оборудования и протоколами
  - **Отдел новых продуктов:** прошивки/ПО ближе к железу (контроллеры и т.п.), ~3 программиста
  - **Малые предметные отделы:** внутренние/нишевые инструменты, дизайн интерфейсов
- **Кому служит:** объекты заказчиков (сервер/клиент на объекте); проектные и инженерные команды; развитие собственного оборудования
- **Доля в компании:** разработка — вспомогательное, но критичное звено цепочки проекта; не главное направление по численности/фокусу
- **Как воспринимается руководством:** _уточнить_
