# Структура и работа компании

> Описание факта: чем занимается фирма, отделы, продукты.  
> **Правило:** всё обсуждённое фиксируется здесь. Непроговорённое — пометки «позже» / «ещё не описано». Перестройка процесса — после полной картины.

---

## 1. Направление деятельности

| | |
|---|---|
| **Отрасль** | Автоматизация управления инженерными системами зданий, сооружений, территорий |
| **Текущая деятельность** | Реализация проектов по автоматизации объектов |
| **Продукт на объекте** | **СУ** = **СУ-А** + **СУ-П** |
| **Инструменты для проектов** | **СУ-ПИ** (в т.ч. **СУ-ПИ-TS**) |
| **Масштаб фирмы** | 50+ сотрудников |
| **Роль разработки ПО** | Не главное направление бизнеса, но критичное звено |

**Цепочка проекта:**

```
Заказчик / объект
  → оП: кастомизация СУ-П + состав СУ-А под заказчика
  → оИ: пуско-наладка на объекте
  → СУ на объекте = СУ-А (Соб + Стор) + СУ-П
```

**оП** и **оИ** работают над проектами с помощью **СУ-ПИ**.

## 2. Основные отделы

Вспомогательные отделы не рассматриваем. Сумма: **35 чел.**

| ID | Отдел | Функция | Чел. |
|----|-------|---------|------|
| **оП** | Проектный | Кастомизация **СУ-П**, определение состава **СУ-А** под заказчика; пользователь **СУ-ПИ** | 8 |
| **оИ** | Инженерный | Пуско-наладка СУ на объекте; пользователь **СУ-ПИ** | 8 |
| **оРПО** | Разработка ПО — Основной | **СУ-П** (**СУ-П-бэк**, **СУ-П-фронт**) + **СУ-ПИ-TS**; новый руководитель; в основном не ИИ-оптимисты | 8 |
| **оРПИ** | Разработка ПО — Инструментальный | _(функция ещё не описана)_ | 1 |
| **оРПBIM** | Разработка ПО — BIM | _(функция ещё не описана)_ | 1 |
| **оРПБИС** | Разработка ПО — БИС | _(функция ещё не описана)_ | 2 |
| **оРУ** | Разработка устройств | **СУ-А-Соб**: **СУ-А-R**, модули (**mDALI**, **AI/AO/DI/DO**, …) — железо + прошивки | 7 |

## 3. Продуктовые сущности

```
СУ — Система управления (на объекте у заказчика)
├── СУ-А — аппаратная часть
│   ├── СУ-А-Соб — собственная разработка и производство (+ прошивки)  ← оРУ
│   │   ├── СУ-А-R — центральный контроллер
│   │   └── модули расширения (СУ-А-R работает с оборудованием в основном через них):
│   │       ├── СУ-А-mDALI — DALI (освещение)
│   │       ├── СУ-А-AI — аналоговые входы
│   │       ├── СУ-А-AO — аналоговые выходы
│   │       ├── СУ-А-DI — дискретные входы
│   │       ├── СУ-А-DO — дискретные выходы
│   │       └── … — другие модули расширения (описать позже)
│   └── СУ-А-Стор — сторонних производителей
└── СУ-П — программная часть (на объекте)  ← оРПО
    ├── СУ-П-бэк — сервер
    └── СУ-П-фронт — клиентское приложение

СУ-ПИ — программные инструменты (для оП и оИ)
└── СУ-ПИ-TS — базовый инструмент (давно разработан и развивается)  ← оРПО
```

| Код | Что это | Где | Кто |
|-----|---------|-----|-----|
| **СУ** | Система управления | объект заказчика | внедряют **оП** + **оИ** |
| **СУ-А** | Аппаратная часть СУ | объект | = Соб + Стор |
| **СУ-А-Соб** | Своё железо + прошивки | объект | разрабатывает **оРУ** |
| **СУ-А-R** | Центральный контроллер | объект | **оРУ**; с оборудованием — в основном через модули расширения |
| **СУ-А-mDALI** | Модуль расширения DALI (освещение) | объект | **оРУ** |
| **СУ-А-AI** | Модуль расширения: аналоговые входы | объект | **оРУ** |
| **СУ-А-AO** | Модуль расширения: аналоговые выходы | объект | **оРУ** |
| **СУ-А-DI** | Модуль расширения: дискретные входы | объект | **оРУ** |
| **СУ-А-DO** | Модуль расширения: дискретные выходы | объект | **оРУ** |
| **СУ-А-…** | Другие модули расширения | объект | **оРУ**; перечень — позже |
| **СУ-А-Стор** | Железо сторонних производителей | объект | состав определяет **оП** |
| **СУ-П** | ПО системы управления | объект | разрабатывает **оРПО**; кастомизирует **оП** |
| **СУ-П-бэк** | Сервер СУ-П | на **СУ-А-R** или на другом ПК с Linux | **оРПО** |
| **СУ-П-фронт** | Клиентское приложение СУ-П | ПК или мобильное (Win, Linux, Android, iOS) | **оРПО** |
| **СУ-ПИ** | Инструменты для работы над проектами объектов | у **оП** / **оИ** | **СУ-П ≠ СУ-ПИ** |
| **СУ-ПИ-TS** | Базовый инструмент СУ-ПИ | у **оП** / **оИ** | развивает **оРПО** |

**Не входит в СУ как наше изделие:** конечные устройства объекта (светильники, вентустановки и пр.).

**Граница:** прошивки устройств — часть **СУ-А-Соб**, не **СУ-П**.
